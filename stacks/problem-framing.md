# Stack: Problem Framing

Use when the task is vague and you need a clean starting point.

Useful inputs:

- the raw request, problem statement, or situation
- any visible constraints or deadlines
- any facts you already know versus what still feels unclear

Suggested blocks:

1. `mode.explore`
2. `core.frame.task`
3. `strategy.problem_split`
4. `core.scope.frame`
5. `core.guardrail.uncertainty`

Expected outcome:

- clear problem frame
- subproblems
- open questions
- best next step
