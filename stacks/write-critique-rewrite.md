# Stack: Write, Critique, Rewrite

Use when the first draft should be fast but the final draft should be sharp.

Useful inputs:

- the intended audience and goal
- source points, notes, or a rough draft
- any constraints on length, tone, or format

Suggested blocks:

1. `core.brief-to-draft`
2. `mode.critique`
3. `core.critique-argument`
4. `core.rewrite-for-clarity`
5. `rubric.writing-quality`

Expected outcome:

- first draft
- prioritized critique
- stronger revision

Domain tags:
- writing
- editorial process
- quality improvement
