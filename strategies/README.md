# Strategies

Strategies are reusable method blocks.

Use a strategy block when the problem needs a specific cognitive move. Do not stack them automatically; each one should earn its place.

## Available Strategies

- **Problem Split** - decompose a tangled task before acting
- **Premortem** - assume failure and reason backward before commitment
- **Steelman** - argue the strongest opposing case before critiquing
- **Inversion** - define the bad outcome, then design to prevent it
- **Red Team** - argue against your own plan as a hostile critic

## When To Use One

- breaking a problem apart -> Problem Split
- pressure-testing a plan before commitment -> Premortem, Inversion, Red Team
- challenging a position rigorously -> Steelman
- surfacing assumptions and failure modes -> Premortem + Inversion together

Use the smallest number of strategies that materially improve the task.
