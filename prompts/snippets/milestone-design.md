# Milestone Design

Use when you need to define meaningful checkpoints for a plan or project — ones that test real progress, not just activity.

```text
Design milestones for the following initiative.

Initiative: {initiative}

Desired outcome: {outcome}

Rough timescale: {timescale}

For each milestone, specify:
- Name — an outcome-oriented label, not an activity label
- Definition of done — exactly what must be true for this milestone to be complete
- Leading indicators — early signals that suggest you are on track before the milestone is reached
- Key risks — what most commonly causes this type of milestone to slip
- Decision point — what question does reaching this milestone answer about whether to continue?

Aim for 3–5 milestones. The final milestone should match the desired outcome directly.
```
