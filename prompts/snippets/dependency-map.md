# Dependency Map

Use when you need to understand sequencing, critical paths, and parallelisation opportunities in a plan.

```text
Map the dependencies in the following work.

Work to be sequenced: {work_description}

Produce:
1. Dependency list — for each task or workstream, what must be complete before it can start
2. Critical path — the longest chain of dependent steps that sets the minimum delivery time
3. Parallel tracks — work that can proceed simultaneously without blocking other tasks
4. Bottlenecks — tasks that block disproportionately many downstream items
5. External dependencies — anything awaiting input, approval, or delivery from outside the team
6. Sequencing recommendation — the order that minimises risk and reduces idle time

If the work is underspecified, state what information you need before mapping it reliably.
```
