# Gap Analysis

Use when you need to identify what is missing — in a plan, argument, knowledge base, or coverage area — relative to what is needed.

```text
Perform a gap analysis on the following.

Subject: {subject}

What it needs to cover or achieve: {requirements}

Current state or coverage: {current_state}

Identify:
1. Critical gaps — missing elements that would cause failure or significant harm if unaddressed
2. Coverage gaps — areas present but insufficiently developed
3. Assumption gaps — things being taken for granted that are not actually established
4. Dependency gaps — things that cannot proceed until something else is in place
5. Nice-to-have gaps — improvements that would add value but are not essential

For each gap:
- Name it precisely
- Explain the consequence of leaving it unaddressed
- Suggest what addressing it would require

Prioritise the list: which two gaps most need attention first and why?
```
