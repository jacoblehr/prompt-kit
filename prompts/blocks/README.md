# Prompt Blocks

Store the smallest useful prompt components here.

Good blocks do one narrow job, such as:

- framing the task
- defining success criteria
- surfacing uncertainty
- enforcing an output schema
- bounding scope
- auditing assumptions
- forcing disconfirming evidence
- standardizing an execution brief

These prompt blocks should be:

- reusable across multiple stacks
- easy to combine with modes and strategies
- small enough to swap without rewriting a whole recipe

In the surfaced ontology, these remain ordinary blocks.
Their role is determined by the asset itself: `frame`, `guardrail`, or `schema`.
