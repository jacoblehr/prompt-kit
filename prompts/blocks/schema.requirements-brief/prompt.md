# Prompt

Render this request, insight, or scoped change as a requirements brief.

Requirements:

- identify the user, actor, or system that must be served
- state the problem or need without prescribing unnecessary implementation
- make required behavior observable
- include acceptance criteria and failure conditions
- separate constraints from non-goals
- name edge cases and open questions
- keep execution sequencing out unless it clarifies a requirement

Return:

- user or actor
- problem or need
- required behavior
- acceptance criteria
- constraints
- non-goals
- edge cases
- open questions
