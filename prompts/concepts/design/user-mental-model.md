# User Mental Model

Apply when designing or improving a product, communication, or system for human use, to surface the gap between how users think and how the system actually works.

```text
Map the user mental model and its gaps for this system or communication.

Cover:
- how users most likely conceptualise this system (their internal working model)
- where that mental model differs from how the system actually works
- which mismatches will cause the most friction, errors, or abandonment
- what vocabulary, metaphors, or analogies users are likely to bring in
- what prior experience shapes their expectations

Then suggest:
- the two or three design or communication changes most likely to close the most
  damaging gaps
- any terminology that is creating silent confusion and should be changed

System or communication to analyse:
{paste product, interface, message, process, or feature description}
```
