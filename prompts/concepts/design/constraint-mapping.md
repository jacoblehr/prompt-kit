# Constraint Mapping

Apply when designing a solution to surface all constraints, distinguish hard limits from soft ones, and find room for creative moves that are being blocked by unexamined assumptions.

```text
Map the constraints acting on this design problem.

Categorise each constraint:
- hard constraints: non-negotiable (legal, physical, safety, contractual)
- soft constraints: preferences or norms that could be relaxed with justification
- assumed constraints: things being treated as fixed without having been explicitly tested

For each soft and assumed constraint:
- who imposed it and what was the original intent?
- what would actually happen if it were relaxed or removed?
- is there a version of the solution that is better precisely because it challenges
  this constraint?

Identify the one constraint that, if removed, would unlock the most design freedom.
Identify the one assumed constraint most likely to be wrong.

Design problem or situation:
{paste problem description and any stated constraints}
```
